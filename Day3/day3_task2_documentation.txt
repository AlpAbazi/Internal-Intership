Day 3 - Task 2 Documentation

WHAT WAS BUILT:
A C program that takes a start and end value from the user to define a range, then analyzes the numbers in that range to count and sum even and odd numbers, and determines which category has more numbers.

LOGIC USED:
The program uses the following approach:
- Prompt user for start and end integers
- Validate that start <= end; if not, display error message and exit
- Initialize counters (even_count, odd_count) and sums (even_sum, odd_sum) to zero
- Calculate total number of values in the range
- Use a for loop to iterate from start to end (inclusive)
- For each number, use the modulo operator (%) to check if it's even (i % 2 == 0) or odd
- Increment the appropriate counter and add to the respective sum
- After loop completes, compare counts to determine which category has more numbers
- Display all results: interval, total values, counts, sums, and the category with more numbers

HOW THE SOLUTION WORKS:
1. User inputs start and end values
2. Program validates the interval (start must be <= end)
3. For loop iterates through each number from start to end
4. Modulo operator determines even/odd status
5. Running counts and sums are maintained
6. Output displays comprehensive analysis including:
   - The interval range
   - Total number of values analyzed
   - Count of even numbers and their sum
   - Count of odd numbers and their sum
   - Which category has more numbers

TEST RESULTS:
Test Case 1: Small interval (1 to 5)
- Interval: 1 to 5, Total values: 5
- Even: count=2, sum=6 (numbers: 2, 4)
- Odd: count=3, sum=9 (numbers: 1, 3, 5)
- Result: More odd numbers ✓

Test Case 2: Equal start and end (5 to 5)
- Interval: 5 to 5, Total values: 1
- Even: count=0, sum=0
- Odd: count=1, sum=5
- Result: More odd numbers ✓

Test Case 3: Invalid interval (10 to 5)
- Error message displayed: "Invalid interval: start must be <= end."
- Program exits without processing ✓

All test cases passed successfully.
